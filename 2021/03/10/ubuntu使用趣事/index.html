ubuntu使用趣事

本文最后更新于：2021年4月1日 中午

系统：ubuntu18.04

  1. ubuntu自带源更新慢

    把自带源更新了

    1. 文件（sources.list）地址 /etc/apt/sources.list

    2. 备份文件 cp sources.list sources.list_backup

    3. 修改文件 sudo gedit sources.list

    4. 添加阿里源

      deb http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse

    5. 更新 sudo apt-get update && sudo apt-get upgrade

  2. 安装python3.6

    sudo apt install python3.6

    好像ubuntu有自带python3，但是我在 sudo apt-get update之前死活进不了python环境，update之后，才正常进入（养成习惯，每次操作前，都要update一下）

  3. 安装Python虚拟环境

    先安装软件包

    sudo apt install virtualenv

    sudo apt install virtualenvwrapper

    然后去安装，但是！俺发现没有pip3，于是先下载了一下pip3

    sudo apt install python3-pip

    有了pip3我就可以顺理成章下载安装虚拟环境了

    pip3 install virtualenv

    pip3 install virtualenvwrapper

    安装完成后去配置一下环境，进入home目录，找到.bashrc文件

    sudo gedit ~/.bashrc

    在最后面加上三行信息

    export WORKON_HOME=$HOME/.virtualenvs
export VIRTUALENVWRAPPER_PYTHON='/usr/bin/python3.6'
source /usr/share/virtualenvwrapper/virtualenvwrapper.sh

    保存文件，运行一下

    source ~/.bashrc

  4. 创建虚拟环境

    创建命令

    mkvirtualenv 虚拟环境名称

    mkvirtualenv -p python3 虚拟环境名称

    我用的后面一句，然后问题出来了（报错如下）

    The path /home/python3 (from –python=/home/python3) does not exist

    一直表示路径不存在，百度了一下没找到解决方法，然后去找了一下我的大佬室友，把它改成了绝对路径，解决了这个问题

    mkvirtualenv -p /usr/bin/python3 虚拟环境名称

    （要是找不到python3路径可以使用 which python3 进行查找）
